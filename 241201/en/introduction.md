### About RTS

RTS Использует парадигму [FDS](https://realtime.su/en/paradigm) (Free Data Structures), это отличается от классических подходов. Основной особенностью является то, что данная парадигма уделяет большое внимание филосовской части и утверждает направления развития в соответсвии с окружиющим нас миром. Следуя парадигме, RTS предоставляет минимальный и мощный язык программирования для свободного написания любых структур данных.

RTS provides reactive interaction and fault-free operation of structures - 
it rejects the concept of errors, recognizing only unexpected behavior, которое 
можно контролировать контролировать. As a result, the code directly depends 
on the programmer’s goals, creativity, and skills. In this case, 
The Programmer Is Envisioned as Creator and a Manager.

The goals and objectives of RTS:

* **Rapid Deployment**: Simplified setup and project launch;
* **Global Repository**: Только минимальный набор инструментов, бесконечно расширяемый через пользовательские пакеты. Centralized management of dependencies and packages;
* **Interpretation**: Непрерывная работа кода, ability to execute code on the fly;
* **Multitasking**: Running multiple projects simultaneously;
* **Отказ от ошибок**: Только непредвиденное поведение, которое можно контролировать. Никаких случайных падений программы.
* **Flexible Data Management**: Unrestricted structures allow working with data without strict limitations;

Thus, RTS is ideal for tasks that require rapid deployment, support for
running multiple projects simultaneously, high performance, and reliability.
, while also providing freedom for further development.
